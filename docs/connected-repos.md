# Connected Repositories

## Purpose

This document defines how this repository references and interacts with other repositories in the wider ecosystem.

---

## Principles

- Do not invent dependencies
- Treat other repos as companions unless explicitly integrated
- Link for context, not assumption
- Keep this repo self-contained

## When to Link

Link another repo when:
- it shares conceptual territory
- it could consume or produce similar structured outputs
- it represents a related system (analysis, automation, classification, etc.)

---

## When Not to Link

Do not link when:
- the connection is superficial
- it introduces confusion
- no real interoperability exists

---

## Future Integration

If integration becomes real, document:
- shared schemas
- data exchange formats
- API or file interoperability
- cross-repo workflows

Until then, treat links as ecosystem context only.
