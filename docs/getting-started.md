# Getting Started

Right. Here’s the version you actually need.

## Step 1 — Load the skill
Use the **strict** skill file first:

`skill/deweygraph-file-organizer.strict.skill.md`

It’s less likely to wander off into philosophy when you just want something sorted.

---

## Step 2 — Pick a folder
Pick one. Not five.

Good choices:
- Downloads
- one project folder
- one archive you vaguely understand

Bad choices:
- your entire home directory
- external drives labelled “old stuff”
- anything containing 15 years of “misc”

---

## Step 3 — Choose a prompt

- want everything → `prompts/full-pipeline.md`
- want notes → `prompts/obsidian-ingest.md`
- want to clean chaos → `prompts/duplicate-audit.md`

---

## Step 4 — Run it
Paste the prompt. Point it at the folder.
Let it analyse and propose.

Important: it is proposing, not acting.

---

## Step 5 — Review before doing anything
Look at:
- folder structure suggestions
- duplicate recommendations
- naming changes

If something looks wrong, it probably is.
Fix the logic, not the symptom.

---

## Step 6 — Export to Obsidian
Take the generated notes and:
- drop them into your vault
- keep the structure consistent
- do not randomly rename things halfway through

---

## Step 7 — Maintain the register
When you hit a weird edge case:
- write a rule in the Decision Register

Otherwise the system will forget and contradict itself later.

---

## That’s it

If you try to be clever and scale this immediately, you will create a very tidy version of chaos.

Start small.
Then scale.
