# Classification Manual (Condensed)

## Core Rule
One item, one anchor.

## Anchor Selection
Choose based on dominant function, not incidental themes.

## Secondary Domains
Use for meaningful cross-domain relevance only.

## Lens Tags
Describe perspective, not subject.

## Relations
Must be typed and reversible.

## Collections
Treat folders as single units when context matters.

## Edge Cases
- notebooks -> classify by subject
- hybrid projects -> choose dominant domain, link the rest
- symbolic technical work -> do not re-anchor unless symbolism dominates

## Decision Notes
Always explain difficult choices.

## Register
Recurring rules must be stored in the decision register.
