# Notebook Rules

Notebooks are where classification systems go to get confused.

## Core problem

A notebook can contain:
- technical diagrams
- writing
- lists
- fragments
- noise

Treating it as one clean subject is often wrong.

## Rule

Classify notebooks by **dominant role in the archive**, not internal content mix.

## Cases

- project notebook → anchor to project or 700.STUDIO
- lab notebook → 621.3.CIRC or relevant domain
- literary notebook → 821.POET
- personal journal → 920.MEM

## When ambiguous

If no clear dominance exists:
- treat notebook as parent collection
- do not over-split prematurely

## Principle

A notebook is a container first, content second.
