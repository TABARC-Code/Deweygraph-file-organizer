# The Short, Blunt Guide

If everything above felt like a lecture, read this instead.

## What this thing does

It looks at a messy folder and:
- works out what’s in it
- suggests a cleaner structure
- classifies useful items
- gives you notes you can drop into Obsidian

That’s it.

---

## How to use it

1. Pick a folder
2. Use the full-pipeline prompt
3. Read what it says
4. Don’t blindly follow it
5. Apply changes yourself

---

## What not to do

- don’t run it on everything at once
- don’t delete files just because it says “duplicate”
- don’t ignore decision notes
- don’t rename things mid-process

---

## One rule that matters

Every item gets one home.

Everything else is linking.

If you break that, the whole system turns into tagging with extra steps.

---

## If it feels wrong

Good. That means you’re thinking.

Fix the rule, write it in the register, carry on.

---

## If it feels easy

You’ve probably misunderstood it.
