# Full Pipeline Prompt

Use **DeweyGraph File Organizer** in **FULL_PIPELINE** mode.

Scan the target folder and do the following in order:

1. analyse the current structure
2. identify clutter, duplicates, weak naming, archive candidates, and project boundaries
3. propose a cleaner folder structure
4. classify meaningful items using one anchor code each
5. assign secondary domains, lens tags, and typed bi-directional relation links
6. create parent and child notes where useful
7. generate Obsidian markdown notes
8. generate a graph map of the key relationships
9. consult and update the **DeweyGraph Decision Register** whenever a recurring judgement rule appears

If content cannot be inspected directly, classify provisionally and state uncertainty clearly.
Prefer archive recommendations over deletion where uncertainty exists.
